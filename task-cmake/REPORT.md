# Отчёт:

## Сравнение

| Характеристика | Рекурсивный | Итеративный |
|---|---|---|
| Временна́я сложность | O(2ⁿ) | O(n) |
| Пространственная сложность | O(n) — стек вызовов | O(1) |
| `fibonacci(10)` | ~177 вызовов | 9 итераций |
| `fibonacci(50)` | ~4×10¹⁰ вызовов | 49 итераций |

## Почему итеративный быстрее

Рекурсивный алгоритм **многократно пересчитывает одни и те же значения**. 

Итеративный алгоритм проходит по числам **ровно один раз**, сохраняя только два предыдущих значения. Дополнительной памяти (кроме трёх переменных) не требует, накладных расходов на вызов функций нет.
